    7.1 无线与移动网络概述

    无线：通过无线链路进行通信
    移动：处理改变网络接入点的移动用户
    无线网络的模式：
    • 基础结构模式：基站将移动终端连接到有线网络
        • 切换：移动终端更换基站，提供有线网络连接
    • 特定网(ad hoc)模式：无基站，结点只能传输到无线链路覆盖范围内的其他结点
        • 结点将自己组织成一个网络（自组网）：在结点之间路由

    7.2 无线链路与网络特性

    SNR（Signal-to-Noise Ratio，信噪比），BER（Bit Error Rate，误码率）

    7.3 无线局域网：IEEE802.11

    IEEE802.11

    所有都使用 CSMA/CA 进行多址访问，并具有基站和点对点网络版本
    • 频谱划分为不同频率的信道
      • 每个AP选择一个频率（信道）
      • 存在干扰可能：相邻的AP可能选择相同的信道
    • 主机：必须与某个AP关联
      • 扫描信道，监听包含AP名称（服务集标识符SSID）和MAC地址的信标帧
      • 选择一个AP进行关联
      • 可能需要进行身份认证
      • 典型情形：运行DHCP获取IP地址等信息
    多路访问控制：CSMA/CA
    • DIFS 是普通数据帧（即通过 DCF 分布式协调功能）竞争信道时使用的帧间隔。
    • SIFS 是最短的帧间间隔，用于高优先级的帧之间的传输。
    基本思想：允许发送端“预约”信道，而不是随机发送数据帧，从而避免长数据帧的冲突
    • 发送端首先利用CSMA向BS发送一个很短的RTS（request-tosend）帧，RTS帧仍然可能彼此冲突，但RTS帧很短
    • BS广播一个CTS（clear-to-send）帧作为对RTS的响应，CTS帧可以被所有结点接收：消除隐藏站影、发送端可以发送数据帧、其他结点推迟发送
    利用很小的预约帧彻底避免了数据帧冲突

    7.6 移动IP

    RFC 3344：数据报的间接路由、代理发现、向归属代理注册
    代理通告：外部/归属代理通过广播 ICMP 消息（类型字段 = 9）来通告服务
    • 移动节点 (Mobile Agent, MA)
        • IP：128.119.40.186（归属地址，固定不变）
        当前位于访问网络：79.129.13.0/24
    • 外部代理 (Foreign Agent, FA)
        • COA（Care-of Address）：79.129.13.2
        负责为移动节点提供连接入口
    • 归属代理 (Home Agent, HA)
        • 地址：128.119.40.7
        位于归属网络，负责代表移动节点接收数据并转发
    1. ICMP Agent Advertisement（代理通告）
        2. 外部代理（FA）定期在访问网络中广播 ICMP Agent Advertisement 消息（类型字段 = 9），告诉网络中的设备：
        “我是一个外部代理，我的 COA 是 79.129.13.2。”
    1. Registration Request（注册请求）
        2. 移动节点（MA）向外部代理（FA）发送一个注册请求，内容包括：
        字段
        含义
        COA: 79.129.13.2
        表示移动节点现在的转交地址
        HA: 128.119.40.7
        归属代理地址
        MA: 128.119.40.186
        移动节点的永久地址
        Lifetime: 9999
        注册的有效期（单位秒）
        Identification: 714
        防止重放攻击的标识
        Encapsulation format
        指定隧道封装方式（例如 IP-in-IP）
        FA 收到后，将此注册请求转发给 HA（归属代理）。
    1. Registration Request → Home Agent
        2. 外部代理将移动节点的请求转发给 归属代理 (HA)。
        归属代理根据请求：
        • 验证移动节点的身份；
        • 更新自己的绑定表（Binding Table）：128.119.40.186 → 79.129.13.2 ，表示移动节点现在通过外部代理可达。
    1. Registration Reply（注册应答）
        2. 归属代理（HA）处理完成后，发送一个 Registration Reply（注册应答） 消息回给外部代理（FA），内容包括：
        字段
        含义
        HA: 128.119.40.7
        归属代理
        MA: 128.119.40.186
        移动节点
        Lifetime: 4999
        实际分配的注册有效期
        Identification: 714
        与请求对应的标识
        Encapsulation format
        确认使用的隧道封装方式
    1. 外部代理 → 移动节点
        2. 外部代理收到应答后，将其转发给移动节点（MA）。
        至此，注册流程完成
    注册成功后：
    • 归属代理（HA）知道移动节点的新位置；
    • 所有发往移动节点（128.119.40.186）的数据包都会：
        1. 先到达 HA；
        1. HA 使用隧道封装，将包转发给 COA（79.129.13.2，即外部代理）；
        1. 外部代理再转发给移动节点。

    7.7 移动通信网络的移动管理（了解即可）

    7.8 无线与移动对高层协议的影响

    从逻辑上讲，影响应该是最小的
    • 尽力而为（best effort）服务模型保持不变
    • TCP 和 UDP 可以（并且确实）在无线、移动环境中运行
    但在性能方面：
    • 由于比特错误（被丢弃的数据包、链路层重传的延迟）以及切换损失导致的数据包丢失/延迟
    • TCP 将丢失解释为拥塞，会不必要地减小拥塞窗口
    • 对实时流量的延迟损害
    • 无线链路的带宽是稀缺资源
